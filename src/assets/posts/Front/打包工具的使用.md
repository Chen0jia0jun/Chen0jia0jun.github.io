# 打包工具的使用

早期使用的模块方法是```AMD```，随后ES6推出了```ESM```模块化方法，浏览器随后原生支持ESM，但在ESM出来前，node就推出了```CJS```模块化方法，虽然v12后向下兼容了ESM，但不是原生支持，需要在```packjson```中添加：```{ "type": "module" }```,因此打包工具就很有必要了，可以将代码打包/编译为运行环境下所支持的格式，完全兼容各种环境。



但打包工具肯定不止这个作用，主要作用有：

1. 模块化整合，解决依赖管理问题

2. 代码优化，提升生产环境性能
3. 跨环境兼容，适配运行载体
4. 统一管理静态资源，简化工程维护
5. 衔接开发与生产，提供工程化能力（热更新、环境变量注入用以区分开发\测试\生产环境的配置）
6. 减少网络请求 / 加载耗时，提升用户体验

## 通用概念

入口（Entry）：项目打包的起始文件（如`src/index.js`），打包工具从这里开始解析依赖图。

出口（Output）：打包产物的输出目录和文件名（如`dist/main.js`），配置打包后文件的存放位置。

加载器（Loader）：用于**处理非 JS 资源 / 非标准 JS 代码**（如 CSS、图片、TS、JSX），因为打包工具默认只认识 JS，Loader 的作用是将非标准文件转化为工具可识别的 JS 模块。

插件（Plugin）：用于**扩展打包工具的功能**（如生成 HTML 文件、清理输出目录、热更新、压缩代码），Loader 处理 “文件转化”，Plugin 处理 “流程扩展”，二者是打包工具的核心扩展体系。

依赖图（Dependency Graph）：打包工具解析的项目所有模块的依赖关系树，是自动合并、加载模块的基础。

Tree-shaking：基于 ES6 模块静态分析的无用代码剔除技术，理解其工作原理（仅支持 ES6 Module，因为 CommonJS 是动态导入，无法静态分析）。

代码分割（Code Splitting）：将打包产物拆分为多个小文件，实现按需加载（如首屏加载核心代码，其他页面代码懒加载），减少首屏加载体积。

开发环境 / 生产环境：理解二者的差异（开发环境需要热更新、源码映射，生产环境需要压缩、混淆、优化），打包工具如何区分并适配配置。
